Description

The GSMST Smash League website was an on-going project during summer 2016 that I built using primarily PHP and SQL. I had started my Super Smash Bros Melee league during spring and had held three tournaments, so I had a significant amount of data on my hands. Since Elliot or I were in contention for being the top player at that point, I wanted to determine the leader statistically. When I completed that (Elliot was leading at the time), I decided to make the website available to the competitors in order to help legitimize the league's power rankings. Thus, what started as a project to calculate and present player elo ratings turned into a fully-fledged league website. After I completed most of the public-facing pages, I began to build a stronger set of admin tools, including better data input pages and revision apps. The analytics pages originated as an admin tool, but turned into a public page when I realized how helpful graphs and matchup data would be to players.

What I Learned

This project was the first time that I worked with databases on such a large and multi-faceted scale. I learned about how tables within a database can be associated in meaningful ways, such as how the table containing match data links to info on the players competing in the player table. This was also the first time that I implemented nested queries in PHP, which is essential to associated data schema. The data entry component gave me a healthy respect of data versioning. The first time I misentered data, I had no system to correct the mistake and because of the nature of elo, I couldn't correct the problem manually. To fix the error, I had to write a new admin tool that refactored all of the player variables up to the point of the error. In the future, I'll implement refactoring before I make a mistake.

Future

Since the summer 2016 season of my league is over, I have time to turn this project into whatever I want. My goal is to generalize the package so that I can distribute it for use by leagues other than mine. This will involve simplifying installation and increasing the friendliness of tools that I had developed for my own use.
